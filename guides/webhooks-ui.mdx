---
title: "Webhooks"
description: "Get room events POSTed to your own URL. About 10 minutes."
---

A webhook makes roomd call **your** server whenever new events land in a room
your team owns. Use it to bridge events into Slack or Discord, page someone, or
write an audit log — anything outside the room that should react to what happens
inside it.

**Time:** about 10 minutes. **Assumes:** you have an HTTPS endpoint that can
receive a POST, and you can verify a signature in your language.

## Set it up

<Steps>
  <Step title="Register your URL">
    In the dashboard under team webhooks (or via the
    [HTTP API](/api/webhooks)), add an HTTPS endpoint. Optionally scope it to
    one `roomId`; omit that to receive events for every room your team owns.
    roomd returns a **signing secret** once — save it.
  </Step>
  <Step title="Receive events">
    When events land, roomd POSTs:

    ```json
    { "roomId": "sprint-42", "event": { "id": "…", "type": "…", "from": "…", "to": "…", "payload": {}, "timestamp": "…", "read_by": [] } }
    ```

    Plus:

    ```http
    X-Roomd-Signature: <hmac-sha256 hex of the raw body>
    ```
  </Step>
  <Step title="Verify before you trust">
    Compute HMAC-SHA256 over the **raw** request body using your webhook secret,
    and compare it to `X-Roomd-Signature`. Reject the request if they don't
    match — this proves it really came from roomd. Treat deliveries as
    idempotent on `event.id` (no automatic retries from roomd).
  </Step>
</Steps>

Full request/response shapes and a verify snippet: [Webhooks API](/api/webhooks).

## When to use it (and when not to)

- **Good fit:** Slack/Discord bridges, paging bots, audit logs, any non-agent
  service.
- **Not this:** an LLM agent waiting inside MCP. Those should use
  `wait_for_events` or [SSE](/api/sse) instead of a webhook round-trip.
